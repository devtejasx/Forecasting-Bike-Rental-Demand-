## Current State

The notebook (`mlassignment (1).ipynb`) has two distinct parts:

1. **Cells 0–13**: a complete, self-contained model comparison pipeline — loads data (or falls back to synthetic demo data), engineers time features, trains 8 regression models, and prints a results table.
2. **Cells 14–38**: markdown "Task/Subtask/Reasoning" cells and code, generated by an AI coding assistant inside Colab, attempting to align the notebook with the methodology of a specific paper — *Jimmy Du, Rolland He, Zhivko Zhechev, "Forecasting Bike Rental Demand"* (a PDF referenced at `/content/Jimmy_Du,_Rolland_He,_Zhivko_Zhechev,_Forecasting_Bike_Rental_Demand[1].pdf`). **This PDF is not included in the repository and was never actually readable during that session** — the notebook's own cells acknowledge this (cell 32/34: *"Due to the inability to access the paper, a direct comparison and analysis of discrepancies are not possible"*). The cells in this section largely just re-run the same 8 models from part 1 multiple times (with one debugging pass to fix a missing `DecisionTreeRegressor` import) rather than actually changing anything to match the paper.

In short: treat cells 0–13 (and the final results table in cell 37) as the real content; cells 14–36 are a failed/abandoned attempt at paper alignment, kept here as-is as part of the assignment history rather than cleaned up.

## Features

- Automatic fallback to a **synthetic demo dataset** (`make_demo_dataframe()`) if no real training data is present, so the notebook is runnable out of the box without a dataset
- Time-based feature engineering from a `datetime` column (`hour`, `weekday`, `month`)
- Automatic removal of `casual`/`registered` columns if present, to avoid label leakage into the `count` target (this matches the well-known Kaggle "Bike Sharing Demand" dataset schema)
- A shared `rmsle()` metric and `make_scorer` wrapper, used consistently across single train/val split and 10-fold cross-validation for every model
- Side-by-side comparison table of validation RMSLE and 10-fold CV RMSLE across all 8 models

## Data

The notebook expects a Kaggle-style "Bike Sharing Demand" CSV with columns: `datetime`, `season`, `holiday`, `workingday`, `weather`, `temp`, `atemp`, `humidity`, `windspeed`, `casual`, `registered`, `count`.

- **No dataset file is committed to this repository.**
- The notebook looks for it at the hardcoded Colab paths `/content/train.csv` (and optionally `/content/test.csv`).
- If those paths don't exist, it automatically generates a 2,000-row synthetic dataset (`make_demo_dataframe()`) with a similar schema and a synthetic sinusoidal demand pattern, so the notebook still runs and produces (non-meaningful) output.

## Models Compared

All models are wrapped in a scikit-learn `Pipeline` with one-hot encoding of categorical time/weather features (plus `StandardScaler` for scale-sensitive models):

| Model | Notes |
|---|---|
| Linear Regression | Baseline |
| Poisson GLM (labeled "Poisson ENet" in the notebook) | `PoissonRegressor` tuned over `alpha` via `GridSearchCV`, 10-fold CV — L2 (ridge) regularization only, despite the "ENet" (Elastic Net) label; see [Known Issues](#known-issues--inconsistencies) |
| Gradient Boosting Regressor | Default hyperparameters |
| PCR (Principal Component Regression) | `TruncatedSVD` (sparse-friendly) + Linear Regression |
| SVR | RBF kernel, `C=10`, `epsilon=0.1` |
| Random Forest | 400 trees |
| Decision Tree | Default hyperparameters |
| Stacking Regressor | Random Forest + Gradient Boosting as base learners, Linear Regression as the final estimator |

## Known Issues / Inconsistencies

1. **The paper-alignment task (cells 14–36) never actually completed.** The referenced PDF was never accessible in that session, so despite the "Align data preprocessing," "Update model implementations," etc. subtask headers, none of the model implementations were actually changed to match the paper — the same code from cells 0–13 is just re-executed.
2. **"Poisson ENet" naming is misleading** — the GLM model is only L2 (ridge)-regularized (`PoissonRegressor`), not a true Elastic Net (which requires an L1 term); scikit-learn's `PoissonRegressor` doesn't support L1 regularization directly.
3. **Colab-specific hardcoded paths** (`/content/train.csv`, `/content/test.csv`, the referenced PDF path) mean the notebook needs path edits to run outside Colab.
4. **No dataset or paper PDF is committed to the repository** — anyone re-running this needs to source their own copy of the Kaggle "Bike Sharing Demand" dataset (and, if pursuing the original task, the referenced paper).
5. **Filename has a stray `(1)` suffix** (`mlassignment (1).ipynb`), suggesting it was re-downloaded/re-uploaded rather than renamed intentionally.
6. **Several later cells reference debugging attempts against files that don't exist in this environment** (e.g. `open('/usr/local/lib/python3.10/dist-packages/ipykernel/ipkernel.py')` as a proxy for checking notebook content) — these are artifacts of the AI assistant's failed self-verification attempts, not meaningful code.
7. **No LICENSE, no `.gitignore`, no tests** — expected for a single-notebook coursework assignment.
